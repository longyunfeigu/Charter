方案 C · 底部单一安静入口，上拉菜单

保留“动作在底部”的心智（新用户永远找得到），但把两个带框按钮合并成一个无边框的安静入口 ＋ Add project。它不再和项目卡片抢样式——没有边框、没有卡片底色，只在 hover 时浮现。点击向上弹出丰富菜单（图标 + 标题 + 说明）。

🖱 可交互：点击底部 “＋ Add project” 打开 / 关闭上拉菜单。

设计说明

  • 优点：保留“动作在底部”的稳定心智——不管列表多长、要不要滚动，入口位置永远不变；但视觉重量从“两个带框按钮”降到“一行安静文字”，突兀感消失。
  • 优点：上拉菜单里每项有图标 + 说明，比现在两行裸文字更能帮新用户区分“打开已有文件夹”和“新建 / 克隆”。
  • 代价：比方案 A 多占一条底栏；两个动作都多了一次点击。
  • 适合：希望改动最小、保守演进（位置不变，只做减法与合并）的取向。
