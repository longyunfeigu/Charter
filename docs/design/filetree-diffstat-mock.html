文件树状态徽章 — 新增 vs 修改（± 行数）

把鼠标悬停到任意行上，检验徽章与「+ 附到会话」按钮的共存效果。四个方案数据相同：新增 5 个文件、修改 6 个（含大数字 +1204 −87 与超长文件名）。

实现口径（四个方案共用）
  • 行数来自 git diff --numstat（工作区 vs HEAD，staged+unstaged 合并），跟随现有 watcher 400ms 防抖刷新，无额外轮询。
  • 新增文件（untracked / added）不跑 numstat，直接给「新增」态；语义上 U 和 A 合并为一个「新增」视觉，不再暴露 git 内部区分。
  • 删除文件不在树里出现（文件已不存在），冲突 C / 重命名 R 保持现有红/蓝字母不动。
  • 二进制文件 numstat 返回 -：退化为只显示 M 态（方案 A/D）或「mod」pill（方案 B）。
  • 大数字不缩写（+1204 −87 原样展示），列宽用 tabular-nums 保证对齐。
