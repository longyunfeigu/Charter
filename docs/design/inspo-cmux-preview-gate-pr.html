借了什么、怎么落地 cmux 行为 → Charter 落法（改造点加粗）

cmux → manaflow 原样用户为什么爱Charter 落法
Live Web Preview：内嵌浏览器贴着代码实时显示任务工作区的 dev server；cmux 终端的侧栏也给每个工作区标出监听端口 「代码对了」≠「页面对了」——看得见的验收把视觉回归挡在合并之前 验收门新增预览 tab：只探测任务 worktree 进程树里监听的本地端口（来自任务的 dev/验证命令），iframe 沙箱 + 任务 worktree · 隔离徽章。不自动起服务；没有端口就诚实空态（引导配置 dev 验证命令），非 web 项目该 tab 不出现
Preview comments（manaflow）：在活的部署预览上直接留评论，自动另起一个只修这个问题的 agent run 发现问题的瞬间就是表达问题的最佳时机——不用回提示框重新组织语言、不丢失正看着的位置 圈选即回炉——同一思想，不同边界：不另起新 run，截图 + 选区 + 一句话走同一个任务的 request-fix 回路（复用 PIVOT-020 批注器与 Review v2），agent 带着精确坐标继续修。任务史保持一条线，Replay 不断链
检查/验证在合并前集中展示 合并的信心来自一眼可见的全绿 检查 tab 集中呈现既有验证机制（VER）：失败复跑后旧记录标 superseded 不覆盖（VER-005）、代码再变标 stale（VER-008）、没跑就大字 Unverified（VER-007）——诚实语义一字不改，只是集中了
One-Click PR Creation：工作区里直接开 PR / squash 合并，GitHub Actions / CI 状态应用内可见 从「本地接受」到「团队可见」零摩擦 PR 草稿：接受后生成分支名 + 标题 + 正文（目标 / 变更清单 / 验证矩阵 / Replay 回执哈希——我们独有的可核查证据）。推送与创建永远是用户显式动作（GIT-007：agent 工具层没有 push），按钮只是「用 gh 起草」，不是「已发出」。PR 建好后的 CI 状态只读回显到 Room，可作后续增量
Heatmap Diff Viewer：给「有风险的改动行」做颜色标注，让审查聚焦在要紧的行 审查疲劳是真问题——几百行 diff 里值得盯的是少数 后续候选（本稿未展开）：验收门 diff 加一层「风险热力」——着色依据接工具网关既有的 R 级分类与路径敏感度（可解释、可关），而不是再请一个模型打分

三条设计原则

1 · 预览打的是任务自己的树 NEW源自 cmux

端口探测范围 = 任务 worktree 的进程树，绝不把主树的 dev server 当成任务的效果图。 预览与 diff、检查共享同一个「wt/t-412 → main」语境——三个 tab 是同一份证据的三种投影。

2 · 视觉问题走同一条回炉道 NEW

Review v2 里选中 diff 行可以回炉；现在圈选预览像素也走同一个 request-fix 回路—— 一个反馈机制，两种证据形态。agent 收到的是结构化的选区 + 截图 + 文字，不是「那个提示怪怪的」。

3 · PR 是证据的导出，不是新的自动化 NEW

正文不是让模型再写一遍小作文，而是证据账本的投影：验收条件勾了几条、验证矩阵、Replay 回执哈希。 审 PR 的同事拿到的是可核查的记录——这是 cmux 没有的部分。

不变的部分（安全语义原样）

GIT-007 与接受边界 KEEP

agent 工具层永不提供 git push；接受 = 合并预检 + 字节校验合回主树 + 可回滚， 与今天完全一致。PR 草稿只在 ACCEPTED 之后出现，绝不反过来诱导接受。

验证语义 KEEP

superseded / stale / Unverified 三个诚实标注原样保留；未验证仍可接受但要显式确认。 检查 tab 是呈现层，不改一条状态机规则。

回炉即对话 KEEP

圈选回炉复用 steer 通道：任务回到进行中、Room 时间线多一条你的反馈（带附件）、 review bar 待新一轮 REVIEW_READY 再回来——没有第二套「评论系统」。

刻意不搬的（以及为什么）

自动 push / 自动合 PR SKIP

cmux 可以选完就建 PR；Charter 的边界是「外发动作必须显式」。GIT-007 是硬规则， Full 模式（ADR-0012）的自动接受也只到本地 ACCEPTED 为止，永远不含推送。

云端可分享的预览 URL SKIP · V1

manaflow 的云形态能把预览发给同事；Charter V1 是本地优先——预览只对 localhost， 对外暴露涉及全新的威胁面（隧道、鉴权、录制），值得做但必须是独立 ADR。

预览里嵌 DevTools SKIP

预览负责「看效果、圈问题」；要真调试，一键在系统浏览器打开同一端口。 在 IDE 里再造半个浏览器审查器是无底洞。
