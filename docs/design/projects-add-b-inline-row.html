方案 B · 列表内的“Add project”幽灵行

动作跟着列表走，不再钉死在面板最底端。最后一张项目卡之后是一个虚线幽灵行 ＋ Add project…，与卡片同宽同圆角；点击后原地展开为两个带说明的选项，选完或点 cancel 收起。列表短时动作就近，永远不会孤零零沉在底部。

🖱 可交互：点击虚线行展开选项，Cancel 收起。

设计说明

  • 优点：动作在内容流里，跟最后一张卡片只隔 4px——“项目在哪，加项目就在哪”，视线不用跳到面板底部。
  • 优点：虚线幽灵行是通识的“占位即入口”模式（Notion / Things 的 add row）；空状态时它自然退化成引导卡，不需要单独设计。
  • 代价：列表很长需要滚动时，幽灵行会跟着滚出视野（需要滚到底才能加）；展开会把内容推下去。
  • 适合：项目数量少（≤8，当前也只显示 8 个）、强调内容流连续性的取向。
