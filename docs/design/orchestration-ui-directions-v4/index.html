Orchestration UI / Four directions + synthesis

不要把四块终端
继续塞进聊天页面。

前四个方向分别代表全局协调、顺手观察、深度介入和异常处理；第五个方向把它们收敛成一个完整工作流。点击任一方向进入可交互 Mock。

DIRECTION 01

独立指挥舱

给多 worker 一块真正的工作面：左边选人，中间看一条直播，右边只放决策。适合 4–8 个并行 worker。

新一级页面 全局态势 重度用户
DIRECTION 02

任务页浮动抽屉

聊天页默认保持干净，只留一枚“3 个 worker / 1 个待处理”入口；需要时抽屉覆盖打开，不永久挤压正文。

留在当前页面 渐进披露 轻量协调
DIRECTION 03

单路聚焦剧场

彻底放弃监视墙：任何时刻只看一个 worker，其余 worker 退化成顶部胶片条。适合阅读、接管和精细指挥。

全屏模式 一次看一路 人工介入
DIRECTION 04

事件流收件箱

默认不展示终端，把 worker 输出压缩成“待审批、已完成、被阻塞、改了哪些文件”。终端只是按需展开的证据。

活动页 / 新 Tab 异常优先 异步管理
DIRECTION 05 · RECOMMENDED SYNTHESIS

指挥舱 + 事件语义 + 聚焦下钻

01 作为稳定主架构，04 负责告诉你什么值得关注，03 只在阅读长输出或人工介入时出现。

稳定三栏 事件优先 按需全屏
