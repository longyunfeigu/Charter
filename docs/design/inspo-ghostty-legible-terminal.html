借了什么、怎么落地 Ghostty 行为 → Charter 落法

Ghostty 原样用户为什么爱Charter 落法
OSC 133 语义提示符：⌘↑/⌘↓ 跳上一条/下一条命令，三击选整段输出 十分钟的滚动缓冲不再靠肉眼考古；「上一条命令输出在哪」变成一个按键 块模型：每条命令 = 一个块（命令行 + 输出 + 退出码 + 用时），⌘↑/⌘↓ 块间跳、整块选中；块头悬停出动作：复制输出 / 存为附件 / ⤴ 发给 Room / ↻ 重跑。「发给 Room」复用 @-ref 机制，署名 YOU（TERM-006）
⌃⇧↑/⌃⇧↓（jump_to_prompt）：逐个跳到上一步/下一步提示符——纯键盘，Ghostty 本身没有滚动条标记（1.3 前无滚动条；1.3 起有也是系统原生单色，不按命令结果着色） 不用在滚动缓冲里肉眼找上一条命令在哪，一个键位直接跳过去；至于「哪里失败了」，Ghostty 里还是得一条条跳着看 标记轨（Charter 原创视觉，非 Ghostty 既有 UI）：绿点 = 成功块、红点 = 非零退出码、蓝点 = 运行中、紫点 = 外置会话回合；点标记直达该块并闪烁定位——把 jump_to_prompt 的「键盘可达」升级成「肉眼 + 点击可达」（这层视觉的先行者：VS Code 终端的 overview-ruler 命令装饰——走同一语义提示符谱系的 OSC 633，按退出码着色；更字面的「滚动区打标」先例是 iTerm2 的 SetMark）
进度上报（OSC 9;4）：1.2 起把进度条画在每个分屏顶上（macOS 终端首家），支持 进行中 / 错误 / 不确定 三种语义态 长命令不再是黑箱——不用盯着刷屏猜进度 三层同源进度：坞 tab 进度环 → 状态栏进度条 → macOS Dock 图标进度。没有 OSC 9;4 的命令，识别 vitest/playwright 等已知输出格式做解析降级；都没有则诚实显示「运行中」不装进度。再借 Ghostty 一手：进度过期 + keep-alive（Ghostty 用 14s）——进程崩了，绝不留一根卡死的进度条
命令完成通知（1.3 notify-on-command-finish：never / unfocused / always，动作 bell/notify 可组合，最短时长默认 5s）+ bell 注意力提示 「跑着长命令去干别的」终于安全——完成/失败会来找你 失焦时长命令（>15s，可设）结束发一条系统通知，点击跳回该块首；聚焦时只打 tab 铃铛标记不弹通知。一次边沿一条，延续 PIVOT-014 的通知卫生
cmux（Ghostty 内核的 agent 终端，同一命题的佐证）：需要输入的窗格亮「通知环」、侧栏未读徽章 + 通知面板、⌘⇧U 一键直达未读（聚焦该窗格即自动清除）；触发走 OSC 9/99/777 与 agent hooks 并行跑几个 agent 时「谁在等我」一眼可见——这几乎是 cmux 存在的全部理由 同一条注意力主线不另起炉灶：铃铛/进度环挂在坞 tab 与任务行上，全局未读继续由既有 Inbox / Needs You 承担；外置会话「需要输入」的边沿走同一路径（OSC 9/777 是事实标准，cmux 同用）
失焦分屏调暗（unfocused-split-opacity） 多窗格时焦点永远一眼可辨，误输入归零 坞失焦时整体轻微降亮、编辑器聚焦描边——同一条「焦点确定性」规则同时用于坞 ⇄ 编辑器 ⇄ 速召台（研究稿 ①）

三条设计原则

1 · 终端输出是结构，不是墙纸 NEW源自 Ghostty

块是新的最小单位：跳转按块、选择按块、引用按块、重跑按块。外置 claude / codex 会话里 「回合」即块（沿 ADR-0017 已知 CLI 解析路径）——同一套手势覆盖普通命令与 agent 会话。

2 · 进度只有一个来源 NEW

tab 环、状态栏、Dock 图标画的是同一个数（OSC 9;4 或已知格式解析），永不编造。 解析不到就显示「运行中 · 已用时」，不画假进度条——与「验证未跑就标 Unverified」（VER-007）同一种诚实。

3 · 通知拉你回到「块」，不是回到 app NEW

点通知落点是那条命令的块首（滚动定位 + 闪烁），不是泛泛把窗口带到前台。 与任务通知「点击直达 Room」的既有肌理一致（PIVOT-014），只是粒度从任务细到命令。

不变的部分（安全语义原样）

域分离 KEEP

这一切都发生在用户终端域（TERM-005）：重跑是用户动作、无 agent 审批； agent 命令的输出仍走专用 Output/Test 视图（TERM-006），不会混进坞里冒充手工输入。

通知卫生 KEEP

每个状态边沿一条、聚焦窗口抑制、Settings 可关——命令级通知加入同一套规则与同一个开关面板， 不另起炉灶。E2E 下继续全体静音（PI_IDE_E2E）。

shell 集成可关、可降级 KEEP

集成脚本按 shell 注入（zsh/bash/fish），Settings 一键关闭；关闭或第三方 shell 下， 终端退回今天的纯滚动形态——功能消失但绝不出错，启动环境保持可诊断（TERM-003）。

刻意不搬的（以及为什么）

Kitty 图形协议（终端内渲染图片） SKIP

Ghostty 靠它在终端里显示图片；Charter 是 IDE——图片有真正的查看器与批注器（PIVOT-020）， 在终端里再造一个次等图片通道是重复建设。

GLSL 光标拖尾 / 自定义着色器 SKIP

Ghostty 社区最出圈的玩具，但它服务「终端即个人空间」的情绪；Charter 的房间气质是安静工坊， 光标烟花与之相抵。动效预算已由 Live Board 一处统管（PIVOT-025r），不再添第二个动效系统。
