让人「卧槽」的五个方向

产品已经有四套 skin（Studio / Archive / Terminal / Index），所以这次大升级不能只是第五套配色—— 每个方向都在 结构、材质、活性、叙事 里押注一种「第一眼震撼」。五个 mock 全部使用真实的产品概念 （Session Rail、Needs You、三段门、验证、worktree 隔离、回放），数据是同一个虚构工作日，方便横向对比。

判断框架：「卧槽」从哪来？A 赌的是 活性（一屋子智能体在干活的画面）；B 赌 反潮流的工艺（全行业深色赛博，我们做精装刊物）； C 赌 材质（最上镜的玻璃与光）；D 赌 独有能力（竞品没有全程记录，做不出可拖动的时间）；E 赌 规模化效率（经营车队的操作系统感）。 它们并不互斥——见页尾的组合建议。
ALauncher

Flight Deck 舰队指挥台态势大屏

打开产品的第一眼 = 指挥中心：五仪表态势带 + 舰队示波器、Needs You 发光通道、 活的会话卡（动作行像 tail -f 滚动、三段门序列灯、验证进度条）。发起区沉底为「发射台」——发起只是一瞬，督战才是常态。

回答：「我的智能体们现在都在干什么？」——把 mission control 从列表升维成态势。 实施代价 M — 复用现有事件流，主要是 Launcher 重排 + 动画预算 打开 mock →
BLauncher

Atelier 印刷工作室light-first 工艺排印

全行业都在做深色赛博玻璃，Charter 反着走：晨报 masthead、巨大衬线 composer、 决定是「头条 + 铅字印章」、写作中的会话是「专栏 + 羽毛笔」、右栏是今日账本与版权页，按钮有活字凸版的按压手感。

回答：「凭什么截图一眼认出是 Charter？」——与所有竞品在气质上彻底分开。 实施代价 M — 是现有 Studio/Archive 气质的极致化，token 体系可承接 打开 mock →
CRoom

Obsidian Glass 黑曜石玻璃dark-first 材质跃升

OLED 纯黑上的三层玻璃（rail / 对话 / 工具画布），界面几乎消失，光只从智能体正在工作的地方发出： 每个会话有光谱色、运行中的工具行有流动光带、diff 刚写下的行有 shimmer、画布「live following」跟着 agent 笔尖走。

回答：「哪个方向的演示视频最炸？」——主流高级感的最优解，最上镜。 实施代价 M+ — 视觉层全面刷新 + blur 性能预算（Electron 需实测） 打开 mock →
DRoom

The Reel 时间胶片可拖动时间的主界面

Room 底部常驻一条时间胶片：章节色带（Plan/Work/Fix/Verify）、事件刻度、里程碑旗标。 拖动播放头，对话与画布同步回到那一刻——回放从「功能」升维成主界面的物理规律。mock 可真实拖动（← → 步进，⏎ 回 LIVE）。

回答：「什么是竞品抄不走的？」——没有全程结构化记录的产品做不出这条轴。 实施代价 L — 但 replay v3 原型已在，等于把既有资产提到台前 打开 mock →
EFleet

Conductor 指挥官终端高密度行式 + 全能 ⌘K

给同时经营 10+ 会话的重度用户：一切都是行——状态灯、实时动作列、三段门微条、等宽 diffstat 严格对齐； j/k 航行、行尾单键操作（a approve / g grant / r review）、多选浮出批量决策条、⌘K 直达任何门。

回答：「并行会话涨到 20 个时产品还成立吗？」——Bloomberg 密度 × Linear 秩序。 实施代价 M — 视图层新增「行看板」，引擎与快捷键系统已具备 打开 mock →

怎么选：一道正餐，两道配菜

立场组合（推荐思路）：先选 气质立场（B 的纸与活字 vs C 的黑与光——二选一，这是品牌级决定）， 再叠 结构升级：A 的态势带/发光通道可以直接进 Launcher，D 的时间胶片可以收进 Room 底部（默认折叠、hover 展开）， E 的行看板作为 Sessions 面板的「密度模式」给重度用户切换。

最小惊艳路径：C（或 B）定材质 → A 的 Needs You 通道 + 会话心跳进 rail → D 的胶片先做只读缩略版。 三步每一步都可独立发布，每一步都有一个可传播的「卧槽点」。

工程红线（不因视觉升级而破）：动画预算（不聚焦即冷却、reduced-motion 全停）、humane language、 无 emoji chrome、审阅门永远显式 —— 五个 mock 都已按这些约束绘制。
