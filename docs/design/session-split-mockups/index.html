会话分栏宽度 · 三个设计方向

现状：对话列与工具画布是固定比例（56/44），只有 Expand 一个开关切到 38/62，且打开 Diff 时会自动切过去——这正是「中间有时候太窄」的来源。 以下三个方向都能解决它，但世界观不同。每个 mock 都可以真实交互，直接拖/点试试。

方案 A · 自由拖拽把手a-free-drag.html

最直觉的答案：分界线就是把手。悬停出现胶囊状 grip，拖到哪算哪；双击复位 56/44；触到最小宽度会有阻尼提示；宽度按 Session 记忆，下次回来还是你的布局。

试：拖动中缝 · 双击复位 · 键盘 ←/→ 微调

方案 B · 磁吸档位b-snap-stops.html

不给无限自由，给三个有名字的好位置：对话优先 62、均衡 50、工具优先 38。 拖动时靠近档位会被磁吸，松手一定落在档上；点分界线上的圆点直达；双击轮换。 与现有 Expand 的两档世界观兼容——等于把两档扩成三档并且可视化。

试：拖动感受磁吸 · 点圆点直达 · 双击轮换档位

方案 C · 焦点跟随c-focus-flow.html

宽度跟着注意力走：在哪边点击/停留，哪边就自动放宽到 62%，动画 240ms。 分界线上只有两个控件：⇄ 立即对调宽窄，图钉冻结当前比例（钉住后自动调整停止， chip 变绿）。「太窄」的瞬间会自愈——你去中间打字，中间就宽了。

试：分别点击左右两边 · 点 ⇄ 对调 · 点图钉冻结
