# Session split-width mockups

讨论用 mock，非产品代码。主题：Session 画布里「对话列 ↔ 工具画布」的宽度分配。

## 背景（现状）

- `session-canvas.css`：`.tr-main` 56% / `.session-tool-canvas` 44%，固定 flex 比例，无拖拽把手。
- 唯一的调节手段是右上角 Expand/Balance，切到 38% / 62% 两档。
- `appStore.openPeek(mode: 'diff')` 会自动置 `sessionToolExpanded: true` —— 打开 Diff
  时右侧自动变宽，是「中间有时候太窄」的直接来源。

## 打开方式

浏览器直接打开 `index.html`（或任一变体）。三个 mock 都是真实可交互的。

## 三个方向

| | A · 自由拖拽把手 | B · 磁吸档位 | C · 焦点跟随 |
| --- | --- | --- | --- |
| 核心动作 | 拖到哪算哪 | 拖动吸附到 62/50/38 三档 | 不手动调，宽度跟随焦点 |
| 复位/切换 | 双击复位 56/44 | 点圆点直达、双击轮换 | ⇄ 一键对调 |
| 记忆 | 按 Session 记住任意比例 | 按 Session 记住所在档位 | 图钉冻结当前比例 |
| 与 Diff 自动扩展的关系 | 手动拖过 = 覆盖自动值 | Diff 打开跳到「工具优先」档 | 天然就是这个模型 |
| 风险 | 像素级微调易失控、比例不可复现 | 自由度受限 | 自动行为可能打扰重度键盘用户 |
| 键盘 | ←/→ 2% 微调 | ←/→ 上下档 | —（Tab 到 ⇄/图钉） |

共同规格（三案一致）：对话列最小 380px、工具画布最小 360px；调整动画 160–240ms
且尊重 `prefers-reduced-motion`；现有 Expand 按钮保留语义（A/B 中等于跳档）。

## 需要拍板的问题

1. 记忆的粒度：按 Session、按项目、还是全局一个值？
2. 打开 Diff 的自动扩展是否保留？（A 案建议：用户拖过之后自动扩展不再覆盖手动值）
3. B 案的档位数值（62/50/38）与命名是否成立？
4. C 案是否作为 A/B 之上的可选增强（设置项），而不是三选一？
