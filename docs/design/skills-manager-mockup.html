Charter — Skills 管理器 + 输入框 “/” 选择器
两个界面都是活的。管理器里点每个 skill 的 Off / Auto 开关、点 Audit 看装之前的审计视图;下面 composer 里点 / 会弹出选择器——注意 被禁用(Off)的 skill 不会出现在选择器里。声明了 disable-model-invocation 的 skill 会带 explicit-only 徽章(开着也不会被模型自动触发)。
1 · Settings → Agent → Skills（可视化管理器）

Skills

Import a folder containing a SKILL.md. Charter copies it into a managed store — it never auto-loads skills from your projects.
Skills can bundle scripts that execute code. Review each one before enabling — every script a skill runs still passes your approval gate (Permission Engine), so a skill can’t run anything you didn’t allow.
两态开关:Off = 禁用(不进提示、不进 “/” 选择器、不加载,一键可再开)· Auto = 启用(模型按 description 需要时自动调用,你也仍能手动 “/”)。少数 skill 作者会在 SKILL.md 里声明 disable-model-invocation(部署/提交这类有副作用的),我们尊重它—— 即使开着也只在你手动 “/” 时跑,并显示 explicit-only 徽章。Audit 展开可见 SKILL.md 与打包脚本;Remove 才是从库里删除(两步确认)。
2 · 输入框 “/” 选择器（只列已启用的 skill）

打 / 或点左下角的 / 图标弹出选择器,列出所有已启用(Auto)的 skill;选中插入一个 /skill:name chip。上面把某个 skill 切到 Off,它会立刻 从这个选择器里消失。有副作用 / explicit-only 的 skill 会带标记提示。
