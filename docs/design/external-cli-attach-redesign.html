外部 CLI 交互重设计 — 原地装饰、意图升格

重设计 ADR-0017 决策 4 的触发与过渡：检测到 claude/codex 会话时不再自动搬动终端，而是在原地滑入一条「会话条」（徽章、快照、实时文件计数、动作）。 搬进右侧竖栏（修正为 600px，容纳 TUI ≥80 列）或打开 Session Room 由用户一键发起。终态与安全语义（EXT 徽章 / 入场快照 / FS 记账 / REVIEW_READY） 与已批准的 external-cli-final-mock.html 完全一致 —— 变的只有「谁来发起搬家」。

为什么不是修修补补 ①

自动升格恰好在你正要打第一条 prompt 的时刻（检测延迟 ≤700ms）把正在输入的终端整体搬走、 底部坞折叠 —— 焦点被偷、空间锚点消失。移动用户正在打字的表面，本身就是错的交互。

为什么不是修修补补 ②

布局突变建立在弱信号上（进程标题 700ms 轮询 + ps 兜底 + 退出去抖）。 信号一抖整个工作台跟着抖。新设计里信号只影响徽章与会话条 —— 布局永远不因检测而变。

为什么不是修修补补 ③

旧竖栏固定 350px ≈ 45 列，而 Claude 的 TUI 需要 ≥80 列才不破版 —— 即使实现完美也是挤坏的。修正：竖栏 600px 起、可拖；「大屏」形态推荐 Room（中央栏天然宽）。

三条设计原则

1 · 永远不动正在输入的表面 NEW

检测的全部 UI 后果收敛为：终端 tab 徽章 ✳ claude EXT、窗格顶部 34px 会话条、一次 toast、树行「外部改动」徽章。零布局变化、零焦点转移 —— 你打的每个键都落在原处。

2 · 搬家 = 用户意图 NEW

会话条提供两个升格出口：⇥ 移到侧栏（600px 竖栏 + changes strip，可拖宽、可 ⇤ 归位）与 ⤢ Room（大屏形态，中央栏天然 ≥80 列）。 偏好项 检测后自动移到侧栏 保留给想要旧行为的人，默认关。

3 · 结束不跳回 NEW

/exit 后：会话条变绿色结束态（✻ 已结束 · N files · Review）， scrollback 摘要行照旧。终端留在用户放它的地方 —— 在侧栏就显示归位按钮，绝不自动再搬一次。

不变的部分（已批准，原样保留）

安全语义 KEEP

EXT·unmanaged 徽章、入场快照（临时 index write-tree）、FS watcher 记账、 字节级回滚、REVIEW_READY 永不自动 ACCEPTED —— 一字不改。

Session Room KEEP

外部任务的 Room 形态不变：终端占中央栏（本来就是用户主动打开的），changes rail、 live-follow peek、Review 入口照旧。

任务与账本 KEEP

外部会话 = 一等任务（external_json）、任务列表/通知/Mission Control 照常呈现。会话结束零改动走 PIVOT-031「Answered」。

随本次一并修复的底座缺陷（任何设计都需要）

xterm 挂载模型 FIX

xterm 6 的 open() 只在首次调用时创建并挂载 DOM，再次调用是 no-op（不搬元素）。现有三处 host.innerHTML=''; term.open(host) 全部换成统一的 mountTerminal(host, item)：首次 open()， 之后 host.appendChild(term.element) 真搬家 + fit() + focus()。

由同一根因连坐的路径 FIX

  • 升格面板空白、无法输入（本次事故，E2E 现红：external-cli.spec.ts:89）
  • 会话结束归位后 active=null，坞里空白且点 tab 也回不来
  • 双终端 tab 切换 A→B→A 后 A 空白
  • ⌘E Home ⇄ Editor 往返后当前终端空白

E2E 补强：升格后向终端敲键必须回显（stdin 达 PTY）；以上路径各一条断言。

边界与暂不做

多会话并存

升格位一次一个（与现状一致）：第二个会话照常获得徽章 + 会话条 + Room 入口， 其「移到侧栏」在占位时置灰（tooltip 说明）。竖栏 tab 化留待后续。

键盘焦点白名单

TUI 要吃掉几乎所有按键（Esc/⌘K…）。终端聚焦时全局快捷键收缩到白名单 + 焦点描边 + 跳出键 —— 方向稿已记录，本轮不做，单独排期。

stdin 桥 / 转写解析

composer→CLI stdin 桥与 transcript 解析（stream-json / app-server）仍按 ADR-0017 记录为终态路径，不在本轮。
