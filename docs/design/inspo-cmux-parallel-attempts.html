并行工位 — 把 cmux 的「多模型并行 + 加冕」移植进 Charter（研究稿 ③）

cmux 的核心主张：同一个任务同时派给多个 coding agent（Claude Code / Codex / Gemini …），各自跑在隔离环境里， 跑完对比、选最好的那份。这解决的是真问题——模型选择焦虑与串行等待。Charter 的底座其实已经就位： 任务级 worktree 隔离（PIVOT-030）、多挂载引擎（PIVOT-029）、并发上限与排队（maxConcurrentRuns）。 缺的只是三件呈现层的事：并行 pill、组卡、对比台。关键改造：manaflow 用 Crown 自动加冕（可人工覆盖）， 我们翻转默认——按证据排序、由人加冕：采纳走既有验收门，REVIEW_READY 永不自动 ACCEPTED。

血统注：cmux 团队后来把这套「并行编排」演进为姊妹项目 manaflow（隔离 VS Code 工作区 · Live Preview · Crown · 一键 PR）， “cmux”之名如今属于他们的 Ghostty 内核 agent 终端（通知环 / 未读徽章，见研究稿 ②）。本稿沿用你的语境统称 cmux，事实以两个仓库为准。

借了什么、怎么落地 cmux 行为 → Charter 落法（改造点加粗）

cmux → manaflow 原样用户为什么爱Charter 落法
同一任务并行派给多个 agent/模型（Claude Code / Codex / Cursor / Gemini / Amp / Opencode…），各自跑在隔离工作区（云端或本地 Docker 的独立 VS Code 实例） 不用赌「哪个模型行」——都试，挑结果；等待从串行变并行 并行 pill ×N：composer 一个开关，扇出 N 份尝试。隔离直接复用任务级 worktree（PIVOT-030）与多挂载引擎（PIVOT-029），不引入容器与内嵌 VS Code；并发走 maxConcurrentRuns 排队，成本 ×N 在弹层明示
任务仪表盘：每个 run 的状态、完成/需要输入的提醒 并行多了之后，「谁跑完了、谁卡住了」必须一眼可见 组卡：mission control 里一张卡三条 lane（驱动 · worktree · 实时动作 · ±diffstat · 状态）。通知按组聚合（「1/3 就绪」「3 份可对比」），不是 ×3 轰炸——延续 PIVOT-014 卫生规则
结果对比 + Crown 评选：跑完自动分析各 run 并给最佳标记，diff 视图里可复核、可人工覆盖 省掉逐份看 diff 的时间，直接从赢家开始 对比台 + 证据排序 + 人加冕：列按证据排（验证矩阵 → 改动面 → 成本），重叠文件可并排深看两种写法。不让 LLM 替你选——「哪份对」的裁决权在验收门（REVIEW_READY 永不自动 ACCEPTED；组模式下 Full 自动接受停用）
选定后一键合并/建 PR，其余 run 关闭 收尾干净，不留一地烂摊子 采纳 = 既有验收门：合并预检（基线冲突检查）→ 接受合回主树（字节校验）；其余尝试归档——worktree 回收、Replay 完整保留（输过的方案也查得回）。PR 草稿见研究稿 ④

三条设计原则

1 · 并行是任务的属性，不是新世界 NEW源自 cmux

cmux 是「为并行而生的独立 app」；Charter 把它收敛成 composer 上的一个 pill。 没有新表面、没有新心智：组卡住在 mission control，采纳走验收门，回滚/Replay/通知全是既有机制。

2 · 证据排序，人来加冕 NEW

对比台不评「谁写得优雅」，只陈列可核查的事实：验证矩阵（含 superseded/Unverified 的诚实标注）、 改动面、重叠写法、成本时长。LLM 评审可以以后作为参考列加入，但裁决永远是人。

3 · 输家也有尊严 KEEP

未被采纳的尝试归档而非蒸发：worktree 回收省磁盘，但事件账本与 Replay 完整保留—— 下周想「当时 sonnet 那份是怎么写的」，还查得回（HIST-001/002）。

不变的部分（安全语义原样）

隔离与合并 KEEP

每份尝试 = 一个标准任务 + 一棵 worktree：权限、工具网关、变更记账全部 per-mount（PIVOT-029/030）。 合并预检、冲突二次确认、字节级回滚，一字不改。

验收门唯一 KEEP

「采纳」没有旁路：它就是那扇既有的门（Review v2 + 验证证据 + 接受/回滚）。 对比台只是把三扇门并排摆开，方便你先挑再进。

并发与成本诚实 KEEP

maxConcurrentRuns（默认 3）之外排队、per-task 命令并发限制（CMD-006）不变； 成本 ×N 在扇出前明示，跑完后每列如实展示 usage。

刻意不搬的（以及为什么）

LLM 自动加冕 SKIP

manaflow 的 Crown 自动颁发（虽可人工覆盖）。默认自动与 Charter 的根性冲突：完成 = REVIEW_READY， 裁决 = 人 + 证据。自动加冕会把「看起来对」加冕成「对」；LLM 评审以后可以作为参考列加入，绝不作为默认裁决。

devcontainer / 云沙箱 SKIP · V1

cmux 每任务起容器换来的是环境隔离；Charter V1 的 worktree + 工具网关 + 权限引擎已覆盖 文件与命令边界。容器化（防 rm -rf 级失控）价值真实，但属于独立 ADR 的大决定，不随并行搭车。

内嵌 VS Code 实例 SKIP

cmux 给每个任务开一个 VS Code；Charter 自己就是 IDE——Room、peek、编辑器已是任务的观察面。 再嵌一个编辑器等于宣布自家编辑器不可信。
